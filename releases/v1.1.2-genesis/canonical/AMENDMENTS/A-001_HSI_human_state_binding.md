# Canon Amendment A-001 — HSI explicit binding under human-state weight = 0

- **Status:** PROPOSED (AI-drafted; pending guardian/steward ratification — the canon
  is substantially single-source and AI holds no authority).
- **Type:** ordinary clarifying amendment (does not alter an Eternity invariant; it
  applies an existing invariant to a named runtime component).
- **Date:** 2026-06-20. **Applies to:** canonical White Paper v1.1.2.
- **Problem solved:** the WP states human-state/biometric weight = 0 categorically
  (36 occurrences) but does not name the **Human State Interface (HSI)** documented
  in `prometheus-happ` (`CONTRIBUTING.md`: `HsiPayload`, axes coherence/affect/
  engagement). This binds that named component to the existing invariant. No
  existing clause names HSI; prefer-amend test: this clarifies, it does not add a
  new invariant.

## Clause text (proposed)
> The Human State Interface (HSI) and any `HsiPayload` or comparable human-
> physiology signal are **research-only** and carry **weight = 0** in PRU, RAP,
> Conf_total, legal gates, governance authority and investor reporting. HSI data
> never functions as an input to value, admissibility, governance, capital or
> public/charitable benefit eligibility, and is never converted into voter, donor
> or campaign-targeting data. Any HSI capture requires explicit consent, data
> minimisation and the human-dignity safeguards of the canon. This clause applies
> the existing human-state invariant to the named component; it creates no new
> capability and authorises no new data use.
